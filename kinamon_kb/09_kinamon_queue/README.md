# 09_kinamon_queue README

> kinamon 宛の全タスク（2択選択、記事承認、プロダクト提案など）を一本化し、パンクを防ぐための司令塔。

---

## 運用ルール

### 1. 優先度フォルダの使い分け
- **`_urgent/` (即対応)**: 24時間以内にチェックが必要なもの（鮮度が命のニュース記事など）。
- **`_normal/` (通常対応)**: 1週間以内にチェックが必要なもの。
- **`_someday/` (いつかやる)**: 面白いアイデア、将来的なプロダクトの種など。

### 2. ルーティン
- 毎朝または毎晩、`_urgent` のリストを確認して消化する。
- 週末に `_normal` を精査し、不要なものは `_someday` へ移動するか削除する。

### 3. タスクの状態遷移
- システムから新しいタスクが発行されると、適切なフォルダに `.md` ファイルが作成される。
- Web UI を通じて「決定」または「保留」を行う。

---

## Web UI（承認システム）連携
- Web UI はこのディレクトリ（`09_kinamon_queue`）及び各 GATE フォルダをスキャンしてタスクリストを表示する。
- **「決定」**: 選択した案に基づき、次工程（SNS記事生成等）へタスクを飛ばし、元ファイルを `_decided` 等に移動する。
- **「保留」**: タスクを `hold` 状態としてマークし、メインのタスクリストからは非表示にする（専用の「保留一覧」からアクセス可能）。

---

## メンテナンス
- タスクが溜まりすぎたと感じたら、評価ボットに優先度の再配分（`priority_control.md`）を依頼するか、手動で `_deprecated` へ移動して「成仏」させる。
